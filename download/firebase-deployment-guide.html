Phase 1

Push Code to GitHub

Upload your project files to a GitHub repository that connects to Firebase.

1Create a GitHub Repository

Go to github.com/new and create a new repository:

  • Repository name: launchfive-studio
  • Description: LaunchFive Studio - Freelancer Team Portfolio Website
  • Visibility: Public (required for free Firebase auto-deploy)

Click "Create repository" and leave this tab open for Step 3.

2Download Your Project Files

From the Z.AI workspace, download the entire project folder as a ZIP archive to your computer. Extract the ZIP to a local folder.

3Upload Code to GitHub

On your new GitHub repo page, click "uploading an existing file". Drag and drop ALL project files from the extracted folder. Add commit message: Initial commit. Click "Commit changes".

Easier Alternative
Install GitHub Desktop (desktop.github.com) to drag-and-drop files. Clone the repo, paste files, click Push.
Phase 2

Create Firebase Project

Set up your Firebase project in the Google Cloud console.

4Open Firebase Console

Go to console.firebase.google.com in your browser. Sign in with your Google account (any Gmail works).

5Create a New Project

Click "Create a project". Fill in:

  • Project name: launchfive-studio
  • Project ID: Auto-generates as launchfive-studio (note this down)
  • Google Analytics: Enable (recommended for visitor tracking)
  • Analytics account: Select Default or create new

Click "Create project". Wait ~30 seconds, then click "Continue".

LaunchFive StudioPage 3
Phase 3

Set Up Firebase Hosting

Connect your GitHub repository to Firebase for automatic deployment.

6Navigate to Hosting

In your Firebase project console, click "Build" in the left sidebar, then click "Hosting". Click "Get started".

7Connect GitHub Repository

Look for "Connect to GitHub" or go to the "Deploy" tab. Click "Connect to GitHub". Authorize Firebase to access your GitHub. Select the repository: launchfive-studio. Click "Connect".

Phase 4

Configure Build Settings

Tell Firebase how to build your Next.js project.

8Set Up Auto-Deploy Configuration

In the Firebase Hosting settings, fill in the build configuration fields:

FieldValue
Branchmain
Build commandnpm run build
Output directoryout
Install commandnpm install

Check "Automatically re-deploy on push" and click "Save".

9Edit next.config.ts for Static Export

Go to your GitHub repo, open next.config.ts, click the pencil icon to edit. Replace the entire file with:

import type { NextConfig } from "next"; const nextConfig: NextConfig = { output: "export", images: { unoptimized: true }, typescript: { ignoreBuildErrors: true }, reactStrictMode: false, }; export default nextConfig;

Click "Commit changes" with message: Configure for Firebase static export.

10Add .nojekyll File

In your GitHub repo root, click "Add file" then "Create new file". Name it public/.nojekyll. Leave content empty. Click "Commit new file".

Why .nojekyll?
This prevents Firebase from ignoring files starting with underscore (like _next/), ensuring all assets load correctly.

Phase 5

Deploy Your Website!

Firebase will automatically build and deploy your site after detecting the code push.

11Trigger the Deploy

After committing the next.config.ts change in Step 9, Firebase auto-detects the push and starts building. Go to Firebase Console → Hosting. You should see a build in progress under "Deploy history". Wait 2-3 minutes for the build to complete. Once done, you will see: Deploy successful!

Your site is now LIVE!
https://launchfive-studio.web.app
Click the URL to see your live website
Phase 6

Connect Custom Domain

Add your own professional domain name with free HTTPS.

12Add Custom Domain in Firebase

In Firebase Console, go to Hosting and click "Add custom domain". Enter your domain (e.g., launchfive.studio). Optionally check "Redirect to www". Click "Continue".

13Update DNS Records at Your Registrar

Firebase will display DNS records to add. Go to your domain registrar (Namecheap, GoDaddy, etc.) and add:

TypeNameValue
TXT@google-site-verification=XXXXX (from Firebase)
A@199.36.158.100
A@199.36.158.101
A@199.36.158.102
A@199.36.158.103

Go back to Firebase and click "Finish". DNS propagation takes 1-24 hours. Firebase auto-provisions a free SSL certificate via Let's Encrypt.

Fix Contact Form with Firestore (Phase 2)

Replace the server API route with Firestore written directly from the browser. No backend needed.

ACreate Firestore Database

Go to Firebase Console → Build → Firestore Database → Create database. Choose "Start in test mode". Select location: asia-south1. Click "Create".

BGet Your Firebase Config

Go to Firebase Console → gear icon → Project Settings → General → Your apps → Web app → Config. Copy the firebaseConfig object.

CUpdate Contact Form Component

Install firebase package and update the contact form to write directly to Firestore. Form data will be saved to a contacts collection, visible in Firebase Console.

import { initializeApp } from "firebase/app"; import { getFirestore, collection, addDoc } from "firebase/firestore"; // Paste your config from Firebase Console const app = initializeApp({ apiKey: "YOUR_KEY", ... }); const db = getFirestore(app); // In your form submit handler: await addDoc(collection(db, "contacts"), { name, email, phone, company, service, budget, timeline, description, referenceLink, createdAt: new Date().toISOString(), });
