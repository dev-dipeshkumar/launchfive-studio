Phase 1

Push Code to GitHub

Upload your pre-configured project files to a GitHub repository.

1Create a GitHub Repository

Go to github.com/new and create a new repository:

  • Repository name: launchfive-studio
  • Description: LaunchFive Studio - Freelancer Team Portfolio Website
  • Visibility: Public (required for free Firebase auto-deploy)

Click "Create repository" and leave this tab open for Step 3.

2Download Your Project Files

From the Z.AI workspace, download the entire project folder as a ZIP archive to your computer. Extract the ZIP to a local folder named launchfive-studio.

3Upload Code to GitHub

On your new GitHub repo page, click "uploading an existing file". Drag and drop ALL project files. Add commit message: Initial commit - Firebase ready. Click "Commit changes".

Easier Alternative
Install GitHub Desktop (desktop.github.com) to drag-and-drop files. Clone the repo, paste files, click Push.
Phase 2

Create Firebase Project

Set up your Firebase project and register a web app.

4Open Firebase Console

Go to console.firebase.google.com. Sign in with your Google account (any Gmail works).

5Create a New Project

Click "Create a project". Fill in:

  • Project name: launchfive-studio
  • Project ID: Auto-generates as launchfive-studio (note this down)
  • Google Analytics: Enable (recommended for visitor tracking)

Click "Create project". Wait ~30 seconds, then click "Continue".

6Register a Web App

On the project overview page, click the Web icon (</>) to add a web app:

  • App nickname: LaunchFive Studio
  • Firebase Hosting: Check "Also set up Firebase Hosting"

Click "Register app". Copy the firebaseConfig object shown on the next screen. Then click "Next" through remaining steps.

Phase 3

Set Up Firebase Services

Enable Firestore for contact form and configure security rules.

7Create Firestore Database

Go to Firebase Console → Build → Firestore Database → Create database. Choose "Start in test mode" (you'll add proper rules in Step 8). Select location: asia-south1 (Mumbai). Click "Create". Wait ~30 seconds for provisioning.

8Set Firestore Security Rules

Go to Firestore Database → Rules tab. Replace the default rules with:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Allow anyone to submit contact form match /contacts/{contactId} { allow create: if true; allow read, update, delete: if false; // Only via Firebase Console } // Block all other collections match /{document=**} { allow read, write: if false; } } }

Click "Publish". These rules allow anyone to submit the contact form, but only you (via Firebase Console) can read submissions.

Phase 4

Add Firebase Credentials

Fill in the environment variables that connect your codebase to your Firebase project.

9Update .env with Your Firebase Config

Go to your GitHub repo, open .env, click the pencil icon to edit. Fill in each value from the firebaseConfig you copied in Step 6:

# Firebase Configuration NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy... NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=launchfive-studio.firebaseapp.com NEXT_PUBLIC_FIREBASE_PROJECT_ID=launchfive-studio NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=launchfive-studio.appspot.com NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789 NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abc123 NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID=G-ABC123XYZ

Click "Commit changes" with message: Add Firebase credentials.

Security Note
These values are safe to commit — they are NOT secret API keys. Firebase security is handled by Firestore Rules + Authentication, not by hiding these values. They are designed to be public.

Phase 5

Deploy Your Website!

Connect GitHub to Firebase Hosting and trigger your first deployment.

11Connect GitHub Repository to Firebase

In Firebase Console, go to Build → Hosting. Click "Get started". Then:

  • Click "Connect to GitHub"
  • Authorize Firebase to access your GitHub account
  • Select the repository: launchfive-studio
  • Click "Connect"
12Configure Build Settings

Firebase will ask for build configuration. Fill in:

FieldValueWhy
BranchmainDeploys from your main branch
Build commandnpm run buildRuns Next.js static export
Output directoryoutNext.js exports to /out with output: "export"
Install commandnpm installInstalls all dependencies including Firebase SDK

Check "Automatically re-deploy on push" and click "Save and deploy".

What Happens
Firebase runs npm install then npm run build. Since next.config.ts has output: "export", this produces a static /out folder that Firebase uploads as your live site.
13Wait for Deployment

Go to Firebase Console → Hosting → Deploy history. Wait 2-3 minutes for the build to complete. You will see: Deploy successful!

Your site is now LIVE!
https://launchfive-studio.web.app

Phase 6

Connect Custom Domain

Add your own professional domain name with free HTTPS.

14Add Custom Domain in Firebase

In Firebase Console, go to Hosting and click "Add custom domain". Enter your domain (e.g., launchfive.studio). Optionally check "Redirect to www". Click "Continue".

15Update DNS Records at Your Registrar

Firebase will display DNS records to add. Go to your domain registrar and add:

TypeNameValue
TXT@google-site-verification=XXXXX (from Firebase)
A@199.36.158.100
A@199.36.158.101
A@199.36.158.102
A@199.36.158.103

Go back to Firebase and click "Finish". DNS propagation takes 1-24 hours. Firebase auto-provisions a free SSL certificate via Let's Encrypt.

src/lib/firebase.ts (Pre-Configured)

import { initializeApp, getApps, getApp } from "firebase/app"; import { getFirestore } from "firebase/firestore"; import { getAuth } from "firebase/auth"; import { getStorage } from "firebase/storage"; const firebaseConfig = { apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY, authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN, projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID, storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET, messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID, appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID, measurementId: process.env.NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID, }; // Initialize once (prevents HMR re-init in dev) const app = getApps().length === 0 ? initializeApp(firebaseConfig) : getApp(); export const db = getFirestore(app); export const auth = getAuth(app); export const storage = getStorage(app);

Contact Form Firestore Write (Pre-Configured)

// Inside ContactForm.tsx onSubmit handler: import { db } from "@/lib/firebase"; import { collection, addDoc } from "firebase/firestore"; await addDoc(collection(db, "contacts"), { name, email, phone, company, service, budget, timeline, description, referenceLink, status: "new", createdAt: new Date().toISOString(), });
